\section{Exemple d'esquema de configuració (Schema-driven JSON)}
\label{annex:schema_json}
A continuació es mostra un exemple simplificat de l'estructura JSON que el \textit{backend} (\textbf{la-core-legacy}) envia al \textit{frontend} per construir dinàmicament la vista de configuració d'una càmera. El format és un diccionari pla on cada clau identifica una propietat del maquinari mitjançant notació de punts, i el valor és un objecte de metadades amb camps prefixats per \texttt{\_} (guió baix). Aquesta convenció és intencional: els prefixos eviten col·lisió de noms amb propietats reals del maquinari, ja que un dispositiu pot tenir propietats com ara \texttt{"type"} o \texttt{"default"} com a dades de configuració pròpies.

\begin{lstlisting}[language=JavaScript, caption={Exemple d'esquema JSON de metadades per a la configuració d'una càmera IP.}, label={lst:schema_json}]
{
  "camera.{id}.name": {
    "_type": "string",
    "_description": "Nom assignat a la camara",
    "_maxLength": 64,
    "_default": "Camera"
  },
  "camera.{id}.enabled": {
    "_type": "boolean",
    "_description": "Activa o desactiva la camara al sistema",
    "_default": true
  },
  "camera.{id}.stream.codec": {
    "_type": "list",
    "_allowedValues": ["H264", "H265"],
    "_description": "Codec de compressio del flux de video principal",
    "_default": "H264"
  },
  "camera.{id}.stream.fps": {
    "_type": "integer",
    "_min": 1,
    "_max": 60,
    "_description": "Fotogrames per segon del flux de video",
    "_default": 25
  },
  "camera.{id}.stream.resolution": {
    "_type": "list",
    "_allowedValues": ["1080p", "720p", "4K"],
    "_description": "Resolucio de captura del sensor de la camara",
    "_default": "1080p"
  }
}
\end{lstlisting}

L'ús de prefixos \texttt{\_} garanteix que les metadades de l'esquema mai solapin amb propietats reals del dispositiu. El component \texttt{ConfigInputComponent} del \textit{frontend} llegeix el camp \texttt{\_type} de cada entrada per decidir quin element de la interfície ha de renderitzar: un commutador per a \texttt{"boolean"}, un selector per a \texttt{"list"} (usant \texttt{\_allowedValues}), o un camp numèric amb rang per a \texttt{"integer"} (usant \texttt{\_min} i \texttt{\_max}).
